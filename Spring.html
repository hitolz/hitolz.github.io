Quick Start

nacosquick start

下载 nacos zip，启动 nacos 服务

sh startup.sh -m standalone

控制台地址，登录名、密码都是 nacos
http://localhost:8848/nacos/index.html#/login

nacos 作为 服务注册中心

服务提供者

pom 依赖

<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>

启动类上添加 @EnableDiscoveryClient

配置

server.port=8070
spring.application.name=service-provider
spring.cloud.nacos.discovery.group=test
spring.cloud.nacos.discovery.namespace=fed092ce-956c-4792-977c-affc5d8f1dc7
spring.cloud.nacos.discovery.server-addr=127.0.0.1:8848

namespace 要注意用 命名空间的 id。

服务消费者

pom 依赖 与 启动类上添加注解与服务提供者一样。

配置

server.port=8080
spring.application.name=service-consumer
spring.cloud.nacos.discovery.group=test
spring.cloud.nacos.discovery.namespace=fed092ce-956c-4792-977c-affc5d8f1dc7
spring.cloud.nacos.discovery.server-addr=127.0.0.1:8848

通过 restTemplate 调用服务提供者的 rest 接口，本例中服务提供者的 url 就是 http://service-provider/xxx/xxx

nacos 作为 配置中心

pom 依赖与作为注册中心的时候不同。

<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>

使用 properties 配置文件

在 nacos 控制台上新加配置

Data ID 命名好后， 配置中的 应用名，及配置的文件格式 都已经确定了。
配置文件格式是 properties
应用名是 nacos-config

项目中要新建一个 bootstrap.properties 的配置文件
bootstrap.properties 中配置：

spring.cloud.nacos.config.server-addr=127.0.0.1:8848

spring.application.name=nacos-config

使用 yml 配置文件

如图新建 yml 类型的配置，
项目中新建 bootstrap-dev.yml 配置文件。

spring:
  cloud:
    nacos:
      config:
        server-addr: 127.0.0.1:8848
        file-extension: yml
  application:
    name: nacos-config2
  profiles:
    active: dev

在项目启动时会到配置中心读取 nacos-config2-dev.yml 的配置

同步刷新

在获取配置的类上 添加注解 @RefreshScope
测试过在启动类上添加不起作用。
